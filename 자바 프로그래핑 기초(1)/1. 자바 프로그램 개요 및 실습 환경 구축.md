## 자바 프로그램 개요 및 실습 환경 구축

1. 자바 언어 개요

   1. 자바 언어의 특징

      자바는 단순하다 - C, C++의 복잡한 기능을 제외하여 코드를 단순하게 작성

      가비지 컬렉터에 의한 자동 메모리 관리로 할당된 메모리 헤제를 신경쓰지 않아도 됨

   2. 자바는 객체지향적이다

      언어라는 객체를 사용하여 프로그램을 작성하도록 하는 언어

      이를 통해 재사용성을 높일 수 있다.

   3. 자바는 분산 처리에 용이하다.

      분산 환경에서 TCP/IP 등의 프로토콜을 통해 효율적으로 실행할 수 있도록 설계된 언어

      서로 다른 컴퓨터 상의 객체들도 원격으로 호출하여 실행할 수 있도록 원격 메서드 호출과 관련된 RMI 라이브러리가 있다.

   4. 자바는 인터프리터에 의해 실행된다.

      인터프리터 언어 - 소스코드 -> 컴파일 -> 중간코드 -> 기계어 해석 -> 실행

   5. 자바는 견고하다.

      다양한 플랫폼에서 실행되기 위해 에러의 원인이 되는 요소들을 제거하거나 자동으로 해결

   6. 자바는 안전하다

      컴파일 시에 엄격하게 데이터 타입을 검사하여 프로그램 실행 시 발생할 수 있는 비정상적인 상황을 미리 방지한다

      클래스 로더 - 어느 컴퓨터에서 온 코드인지 분리한다

      바이트 코드 검증기 - 바이트 코드를 검증하고 문제가 없을 경우 인터프리터에 의해 실행

   7. 자바는 플랫폼 독립적이다

      사용하는 운영체제나 CPU 등의 하드웨어 사양에 관계 없이 실행될 수 있다.

      자바컴파일러와 자바 가상 머신을 사용하여 코드를 실행하기 때문

   8. 자바는 높은 성능을 제공

      가비지 컬렉션 기능이 추가되어 있다

   9. 자바는 멀티스레드를 지원한다

      하나의 CPU로 여러가지 프로그램을 실행하여 실행 시간을 줄여준다

   10. 자바는 동적이다

       자바는 변화되는 환경에 잘 적응되도록 설계

       라이브러리를 코드를 실행할 때 연결한다

       

2. 자바 플랫폼의 종류

   플랫폼 - 프로그램이 실행될 수 있는 하드웨어나 소프트웨어 환경을 의미

   * JAVA SE : 가장 기본이 되는 에디션, 대부분의 패키지 포함
   * JAVA EE : 협업에서 사용되는 API들이 집약, 데이터베이스에 연동되는 많은 기술 포함
   * JAVA ME : 모바일 기기에서 사용되는 API가 포함된 에디션, 현재는 거의 사장

   자바 플랫폼은 운영체제가 아닌 자바 가상 머신에서 실행되는 차이가 있다

   운영체제가 다르면(window, linux) 같은 프로그램이라도 다르게 설계되어야 한다.

   하지만 자바 가상 머신에 의해 운영체제에 영향을 받지 않고 동작할 수 있는 환경을 제공한다. 하지만, 운영체제에 맞는 jvm을 설치해야 정상적으로 동작한다.
